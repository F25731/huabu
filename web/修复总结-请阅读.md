# 🎯 浏览器卡死问题 - 已完全修复

## 核心问题

你说的"登录进去之后直接卡死，图片都没生成，整个浏览器卡死"的真正原因是：

**useEffect 无限循环导致页面疯狂重渲染！**

---

## 🔥 问题根源（最关键）

**位置：** `web/src/components/layout/client-root-init.tsx`

**错误代码：**
```typescript
// ❌ 致命错误：将 zustand store 的函数作为 useEffect 依赖
const hydrateUser = useUserStore((state) => state.hydrateUser);
const loadPublicSettings = useConfigStore((state) => state.loadPublicSettings);

useEffect(() => {
    void loadPublicSettings();  // 调用函数 → 更新 store
}, [loadPublicSettings]);  // 函数引用变化 → 再次触发 → 无限循环 ♾️

useEffect(() => {
    if (!isLoginPage) void hydrateUser();  // 调用函数 → 更新 store
}, [hydrateUser, isLoginPage]);  // 函数引用变化 → 再次触发 → 无限循环 ♾️
```

**循环流程：**
1. 页面加载 → useEffect 执行
2. 调用 `hydrateUser()` / `loadPublicSettings()` 
3. 函数内部调用 `set()` 更新 store
4. store 更新 → 组件重新渲染
5. 函数引用变化 → useEffect 依赖变化
6. useEffect 再次触发 → 回到步骤 2
7. **无限循环 → 页面疯狂重渲染 → 浏览器卡死 💀**

---

## ✅ 修复方案

**修复后的代码：**
```typescript
// ✅ 正确：使用 useRef 标记，确保只执行一次
const hasInitialized = useRef(false);

useEffect(() => {
    if (hasInitialized.current) return;  // 🛡️ 守卫：已初始化则直接返回
    hasInitialized.current = true;

    // 通过 getState() 获取函数，不作为依赖
    void useConfigStore.getState().loadPublicSettings();
    
    if (!isLoginPage) {
        void useUserStore.getState().hydrateUser();
    }
}, [isLoginPage]);  // 只依赖路径，不依赖函数
```

**为什么有效：**
- ✅ `useRef` 的值在整个组件生命周期中保持不变
- ✅ `hasInitialized.current` 确保初始化逻辑只执行一次
- ✅ 通过 `store.getState()` 获取最新函数，避免作为依赖
- ✅ 只在路径改变时才重新评估
