# 登录卡死问题修复说明

## 问题诊断

### 1. **原登录逻辑没有验证API Key**
- 位置：`web/src/services/api/auth.ts` 的 `login()` 函数
- 问题：只是在前端创建虚拟用户，从未真正调用API验证Key是否有效
- 结果：输入任何内容都能"登录成功"，但实际使用时才发现Key无效，导致卡死

### 2. **没有实际的余额查询**
- `balanceStatus` 状态存在但从未真正查询
- 状态一直停留在 `"unknown"` 或 `"checking"`

### 3. **图片生成时卡死原因**
- 位置：`web/src/services/api/image.ts:285-310`
- 问题：轮询等待图片生成结果，如果API Key无效会一直重试直到超时（10分钟）

## 修改内容

### ✅ 1. 添加API Key验证逻辑（`web/src/services/api/auth.ts`）

```typescript
async function validateApiKey(apiKey: string) {
    // 调用 /v1/models 接口验证API Key
    // - 检查401/403（Key无效）
    // - 检查500（服务不可用）
    // - 检查连接超时
    // - 验证返回数据格式
}
```

**验证流程：**
- 调用 `https://api.zmoapi.cn/v1/models` 检查Key有效性
- 10秒超时限制
- 明确的错误提示：
  - "API Key 无效或已失效"
  - "API 服务暂时不可用"
  - "API 连接超时，请检查网络"
  - "无法连接到 API 服务器"

### ✅ 2. 简化余额状态（`web/src/stores/use-user-store.ts`）

按你的要求，不再查询余额，直接设置为 `"available"`：

```typescript

// hydrateUser 时
balanceStatus: "available"  // 恢复会话时也标记为可用
```

### ✅ 3. 优化登录提示（`web/src/app/(user)/login/page.tsx`）

- 说明文字：从"按生图档位填写"改为"**填写知梦 API Key，登录时会验证连接有效性**"
- 成功提示：从"登录成功"改为"**API Key 验证成功，登录完成**"

## 解决的问题

| 问题 | 修复前 | 修复后 |
|------|--------|--------|
| **登录验证** | 不验证，任何输入都能登录 | 真实调用API验证，无效Key会立即报错 |
